Conceptual Question & Answer

Q1: How does JavaScript work?

Ans: JavaScript is a single-threaded, dynamic typing, interpreted programming language. In the browser, JavaScript code is run by a JavaScript engine. Like, Chrome uses V8 engine; Firefox uses SpiderMonkey engine, etc. At first, when the engine gets the code, it converts those codes into AST(abstract syntax trees) through lexical analysis. Finally, the interpreter converts those codes into machine language and js code works.

Q2: What does JavaScript event loop do?

Ans: JavaScript event loop maintains all the event which is happening in JavaScript code. JavaScript executes one script at a time & continues to execute the code one by one. When a function is called to execute, it is added to the call stack. If another function is called from inside of the existing function, then it's added top of the previous function in the call stack. Now, after executing the second function, the stack will pop it out and the first function will be executed and will be popped out. If any function takes too much time to execute then JavaScript would not stop executing other code. It will continue to execute the upcoming script and when the function finishes executing, the result will be shown. After executing all the functions in the call stack, the functions will be popped out and the stack will remain empty. This is what the JavaScript event loop does.
